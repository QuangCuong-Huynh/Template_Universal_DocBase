# Architecture Decision Record

**Domain:** `ADR` &nbsp; **Document ID Pattern:** `ADR-[PROJECT]-[TOPIC]-[DATE]-001`

> Documents a single architectural decision: context, options considered, rationale for the chosen option, trade-offs accepted, and consequences.

## Contents

1. Document Control
2. Context (Problem Statement, Decision Drivers, Constraints)
3. Options Considered (one subsection per option with Advantages/Disadvantages table)
4. Decision (Chosen Option, Rationale, Trade-offs Accepted)
5. Consequences (Immediate Actions, Affected Documents, Risks Introduced, Review Trigger)

**One ADR per decision.** ADRs are never deleted — they form the architectural history.
